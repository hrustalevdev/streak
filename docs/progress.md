### Правки промпта (итеративно, по ходу тестов)

1. Правило 5: запрет вызывать `mark_completion`/`get_streak` с id, не полученным из `list_habits`/`create_habit` в этом же диалоге — иначе модели галлюцинируют id.
2. Правило 6: запрет повторного вызова одного и того же tool с теми же аргументами — иначе `qwen3.5:9b` уходила в цикл (наблюдалось 10 подряд одинаковых `list_habits`).
3. Явное требование: последнее сообщение всегда финальное (без tool_calls) и всегда содержит все 4 поля контракта, ни одно не пропущено.
4. Формат строки `Status` переписан с `Status: success | error` (модель иногда копировала это буквально как значение) на явную инструкцию с примером корректной/недопустимой строки.

### Методика стресс-теста

Один и тот же сценарий из 8 запросов, mock API сбрасывается в чистое состояние перед каждым прогоном на новой модели:
`покажи привычки` → `создай медитация` → `отметь медитацию` → `стрик по медитации` → `что такое streak (без tool)` → `отметь бег (новая)` → `отметь бег (повтор, уже существует)` → `покажи привычки (финал)`.

### Результаты: llama3.1:8b (`http://192.168.1.10:11434/v1`)

| Запрос | Результат |
|---|---|
| покажи привычки (пусто) | ✅ |
| создай медитация | ✅ |
| отметь медитацию | ❌ Нашла привычку через `list_habits`, но `mark_completion` вызвала не с тем id → `habit not found` → вместо повтора создала **дубликат** "медитация" → в финальном ответе `Status: success` с **выдуманным** `Data: habitId="id_новой_привычки"` (буквальный плейсхолдер, не реальный результат) |
| стрик по медитации | ⚠️ Из-за дубликата выше в списке 2 "медитация"; численный streak в `Data` в этот раз присутствует (0) |
| что такое streak | ✅ |
| отметь бег (новая) | ❌ Пропустила `list_habits`, сразу `mark_completion` с несуществующим id → ошибка → `create_habit` только реактивно → так и не отметила выполнение, `Status: error` |
| отметь бег (повтор) | ❌ Опять пропустила `list_habits`, создала **второй дубликат** "бег", `mark_completion` снова мимо → `Status: error` |
| покажи привычки (финал) | ✅ формат, но в данных — 2× "медитация" и 2× "бег" вместо ожидаемых 2 привычек |

**Итог: 3/8 полностью корректно, 2 дубликата привычки, 1 факт фабрикации ответа.** Не рекомендуется.

### Результаты: qwen3.5:9b (`http://192.168.1.10:11434/v1`)

Первый прогон (промпт без правила 6) — «стрик по медитации» ушёл в цикл из 10 одинаковых `list_habits` и закончился **пустым** ответом без контракта. После добавления правил 6 и уточнения формата `Status` — повторный прогон:

| Запрос | Результат |
|---|---|
| покажи привычки (пусто) | ✅ |
| создай медитация | ✅ |
| отметь медитацию | ✅ (`list_habits` → `mark_completion` → `get_streak`, корректный id везде) |
| стрик по медитации | ✅ (1 вызов `list_habits`, полный контракт, streak=1 в `Data`) |
| что такое streak | ✅ (без tool-вызова, полный контракт) |
| отметь бег (новая) | ✅ (`list_habits` → `create_habit` → `mark_completion`, корректно) |
| отметь бег (повтор) | ✅ (нашла существующую через `list_habits`, без дублирования) |
| покажи привычки (финал) | ⚠️ Данные верные (медитация=1, бег=1, без дублей), но формат ответа — markdown-таблица вместо `Status/Action/Data/Errors` (разовый пропуск контракта) |

**Итог: 7/8 полностью корректно (данные и контракт), 1/8 — данные верные, но разовый пропуск формата контракта. Ни одной галлюцинации id, ни одного дубликата.**

### Решение

`qwen3.5:9b` зафиксирована как модель по умолчанию (`LLM_PROVIDER=ollama`, `OLLAMA_MODEL=qwen3.5:9b` в `.env` / `.env.example` / фолбэк в `graph.ts`). `llama3.1:8b` для этого проекта не годится без более серьёзной переработки промпта — слишком часто нарушает обязательный порядок вызовов tools и один раз подставила выдуманные данные вместо реального результата API.

**Остаточный риск:** один разовый пропуск формата контракта у `qwen3.5:9b` на финальном списке привычек — не удалось стабильно устранить дополнительными инструкциями в промпте за отведённое время. Требует либо дополнительного prompt-hardening, либо post-processing/валидации формата на стороне кода (не реализовано).

### Живой тест на локальной llama3.1:8b (замена OpenRouter, 2026-07-17)

Из-за блокера выше правки 1–3 временно проверены через локальную Ollama на домашнем десктопе пользователя (`http://192.168.1.10:11434/v1`, OpenAI-совместимый эндпоинт, `ChatOpenAI` с подменённым `baseURL`/`model` — правка в `graph.ts` не коммитилась, после теста откачена).

| Запрос | Ожидание | Факт |
|---|---|---|
| "что такое streak?" (без tool) | Контракт `Status/Action/Data/Errors` соблюдён (правка 1) | ✅ Формат есть, но `Status: -` вместо `success` — модель не всегда попадает в enum |
| "покажи все мои привычки" | `list_habits`, контракт корректный | ✅ Полностью корректно |
| "создай привычку читать книги" | `create_habit` | ✅ Сработало, но `list_habits` перед созданием не вызван (нарушение правила 1 из промпта) |
| "отметь медитацию выполненной" (привычки нет) | `list_habits` → `create_habit` → `mark_completion` | ❌ Сразу вызвала `mark_completion` с несуществующим id → `habit not found` |
| "отметь привычку читать книги выполненной" (привычка уже существует) | `list_habits` → `mark_completion` с найденным id | ❌ Опять пропустила `list_habits`, `mark_completion` с придуманным id → `habit not found` |
| "какой у меня стрик по чтению книг?" | `list_habits`, streak в `Data` | ⚠️ `list_habits` вызван верно, привычка найдена, но численный streak в `Data` не попал (`Data: читать книги`) |

**Вывод:** правки 1 и 3 корректны на уровне промпта (структура, инструкции), это подтверждено логически и частично — на слабой модели. Но `llama3.1:8b` не проходит собственный тест-сьют репозитория: не соблюдает обязательный вызов `list_habits` перед `mark_completion`/`create_habit` в 2 из 3 релевантных сценариев, и не всегда извлекает нужное значение в `Data`. Это ограничение модели, а не регрессия от внесённых правок — на `openai/gpt-oss-120b:free` (см. `docs/report.md`) все сценарии были задокументированы как корректные до того, как эта модель перестала быть бесплатной на OpenRouter.

**Дальнейшие шаги (открыто):**
- Прогнать полный тест-сьют на целевой модели, когда решится вопрос с OpenRouter (см. блокер выше), либо
- Если переходить на локальную модель — `llama3.1:8b` в текущем виде промпта недостаточно дисциплинирована; нужно либо усиливать промпт (few-shot примеры, более жёсткие инструкции), либо смотреть на `qwen3.5:9b` (тоже есть на домашнем десктопе, заявлен `tools` + `thinking`).
